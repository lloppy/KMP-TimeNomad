# ТЗ: Time Nomad (KMP-клон) — астрология и натальные карты

## 1. Обзор и цель

Клон iOS-приложения **Time Nomad — Astrology Charts**
(https://apps.apple.com/ua/app/time-nomad-astrology-charts/id1092841332) на **Kotlin Multiplatform**
(Compose Multiplatform: Android, iOS, Desktop). Приложение профессионального уровня: точные
астрологические расчёты **на устройстве** (офлайн, без интернета), натальные карты, транзиты,
планетарные часы, профили с заметками и тегами.

Пакет: `com.lloppy.timenomad`. Проект создан на базе шаблона (бывшие «аудио-уроки»): инфраструктура
(Compose, Koin, навигация, тема, Multiplatform Settings) сохранена, бизнес-логика заменена на астрологию.

## 3. Архитектура

- **Модули Gradle:** `shared` (KMP: commonMain/androidMain/desktopMain/iosMain), `androidApp`, `desktopApp`, `iosApp` (Xcode).
- **Слои в `shared/commonMain`:**
  - `astro/` — ядро расчётов (платформонезависимое):
    - `time/` — `AstroTime` (юлианские даты, GMST/LST), `GeoLocation`;
    - `math/` — `AstroMath` (тригонометрия в градусах, наклон эклиптики);
    - `model/` — `CelestialBody`, `ZodiacSign`, `AspectType/AspectHit`, `HouseSystem`, `ZodiacMode`, `Ayanamsha`, `PlanetPosition`, `House`, `Chart`;
    - `ephemeris/` — `EphemerisService` (интерфейс) + `MeeusEphemeris`, `KeplerianPlanets`, `MoonTheory`;
    - `houses/` — `HouseCalculator`; `aspects/` — `AspectCalculator`;
    - `chart/` — `ChartCalculator`, `MoonPhase`; `planetaryhours/` — `PlanetaryHoursCalculator`;
    - `format/` — `AstroFormat`.
  - `data/` — `profile/` (`AstroProfile`, `ProfileRepository`), `settings/` (`AstroSettingsRepository`).
  - `screens/` — `dashboard`, `chart`, `profiles`, `planetaryhours`, `settings` (Screen + ViewModel).
  - `ui/` — переиспользуемые компоненты + `ChartWheel`, `MoonDisk`; `theme/` — палитра, `AstroColors`.
  - `navigation/` — типобезопасные `Destinations` + `AppNavHost` (нижняя навигация).
- **DI:** Koin (`di/Koin.kt`): `appModule`, `astroModule`, `viewModelModule`.
- **Состояние:** MVVM, `ViewModel` + `StateFlow`, Compose `collectAsStateWithLifecycle`.

## 4. Движок расчётов

**Итерация 1 — офлайн, на устройстве, единый код в `commonMain`** за интерфейсом `EphemerisService`:
- **Планеты (Меркурий…Плутон):** кеплеровские элементы JPL/Standish (таблица 3000 BC–3000 AD,
  с поправками b,c,s,f для внешних планет) → гелиоцентрика → геоцентрика.
- **Солнце:** из гелиоцентрического положения Земли.
- **Луна:** ряд Meeus (гл. 47, ELP-2000/82 сокращённый), ~10″.
- **Узел:** средний восходящий узел.
- **Прецессия/аянамша:** перевод J2000 → равноденствие даты; сидерик = тропик − аянамша.
- **Дома:** Asc/MC — точные формулы; Placidus — итерация полудуг; Equal/WholeSign/Porphyry — закрытая форма.
- **Точность:** единицы угловых минут (достаточно для знаков, домов, аспектов).
- **Тесты:** `shared/src/commonTest` — санити-проверки (скорости, элонгации, аянамша, узел).

**Планируемый апгрейд точности (роадмап):** заменить `MeeusEphemeris` на **порт Swiss Ephemeris**
(режим Moshier — без бинарных файлов, чистый Kotlin, ~1″) — drop-in через `EphemerisService`, UI не меняется.

> ⚠️ **Лицензия Swiss Ephemeris:** AGPL-3.0 либо платная проф. лицензия Astrodienst. Для закрытого
> коммерческого приложения требуется платная лицензия. Текущий движок (Standish/Meeus) свободен от этого.

**Альтернатива (если нужен внешний источник):** сетевой `EphemerisService` через Ktor на бесплатном API
(FreeAstroAPI — 80 запросов/день; self-host Kerykeion/Astrologer-API — open-source, ядро Swiss Ephemeris).
Минусы — зависимость от сети и лимиты; противоречит офлайн-принципу оригинала.

## 5. Данные и хранение

- **Профиль (`AstroProfile`):** имя, дата/время рождения, смещение UTC (минуты), широта/долгота, место, заметки, теги.
- **Хранение:** список профилей сериализуется в JSON и сохраняется в Multiplatform Settings (локально).
- **Роадмап:** миграция на SQLDelight (поиск/индексация), опц. синхронизация (iCloud на iOS / своё облако).
- **Геокодинг:** поиск места по названию через **OpenStreetMap Nominatim** (бесплатно, без ключа;
  данные ODbL, требуется атрибуция «© OpenStreetMap» — показана в карточке поиска). Заполняет lat/lon и название.
  Сетевой вызов только при настройке профиля; расчёты карты — офлайн.
- **Часовой пояс/UTC:** оценка по долготе (round(lon/15)), поле редактируется вручную. Роадмап —
  точное автоопределение (база TZ-границ + историч. переходы DST) либо платный TZ-сервис.

## 6. Экраны и UX

- **Дашборд («Небо сейчас»):** фаза Луны (рисованный диск), мини-колесо текущей карты, список положений,
  ретроградные планеты, домашняя локация/время.
- **Карта (`ChartWheel`):** кольца знаков (по стихиям) и домов, глифы планет, линии аспектов (цвет по типу),
  Asc/MC; списки планет (с домом), углов и аспектов (орб, сходящийся/расходящийся). Текущее небо или натал профиля.
- **Профили:** список (аватар, дата, теги), поиск, создание/редактирование (форма данных рождения).
- **Планетарные часы:** управитель дня, текущий час, 24 неравных часа с интервалами.
- **Настройки:** тема (системная/тёмная/светлая), зодиак (тропик/сидерик + аянамша), система домов, домашняя локация.
- **Роадмап:** Транзиты, Синастрия, дневник профиля (датированные заметки), события (затмения, VoC, ингрессии).

## 7. Дизайн-система

Тёмная «звёздная» эстетика как основная: фон `#0B0B14`, поверхности `#14141F`, акценты — золото `#E8C36B`
и бирюза `#3FC9C0`, индиго `#6E63D6`. Светлая тема — опция. Глифы знаков/планет — Unicode (роадмап: векторные
`Path`/астрологический шрифт для единообразия). Цвета стихий/аспектов/планет — `theme/AstroColors`.

> Точные скриншоты App Store через автоматический фетч недоступны; палитра/раскладка воспроизведены по
> описанию и общей эстетике Time Nomad. При наличии официальных скриншотов — уточнить токены и компоновку.

## 8. Платформенные особенности (роадмап)

- **Виджеты:** Android (Glance/AppWidget), iOS (WidgetKit, отдельный таргет) — текущая карта, фаза Луны,
  планетарный час/день, накшатра.
- **Будильник планетарного часа:** Android (AlarmManager/WorkManager), iOS (UNNotification).
- **Локация:** опц. определение по GPS для «неба сейчас».
- **iCloud-синхронизация профилей** на iOS.

## 9. Роадмап по итерациям и критерии приёмки

- **Итерация 1 (выполнено):** очистка шаблона, переименование пакета, офлайн-движок (планеты/Луна/узел/дома/аспекты),
  тропик/сидерик, дашборд, карта-колесо, профили, планетарные часы, настройки; сборка на Android/Desktop/iOS; санити-тесты.
- **Итерация 2:** транзиты (натал ↔ текущее небо), дневник/датированные заметки профиля, события (VoC, ингрессии),
  улучшение Koch/Placidus, верификация позиций против эталона (astro.com / NASA Horizons) с допуском ≤ 1′.
- **Итерация 3:** синастрия/композит/прогрессии, фикс-звёзды, накшатры, апгрейд на Swiss Ephemeris (Moshier-порт).
- **Итерация 4:** виджеты и будильник на Android/iOS, геокодинг+часовые пояса, миграция на SQLDelight, синхронизация.

**Критерии приёмки расчётов:** позиции светил/планет в пределах допуска от эталона; знак/градус, дом и аспекты
совпадают с референсной картой; ретроградность определяется верно; фаза Луны и планетарный час корректны для
заданных даты/локации.

**Команды верификации:**
- `./gradlew :shared:desktopTest` — юнит-тесты движка;
- `./gradlew :androidApp:assembleDebug` — сборка Android;
- `./gradlew :shared:compileKotlinIosSimulatorArm64` — компиляция iOS-таргета;
- `./gradlew :desktopApp:run` — запуск десктопа.
